# Skill Butler - 技能管家

> 个人技能管理元技能 - 统一管理用户级和项目级技能

**版本**: v1.0
**作者**: 阿辰 & Claude

---

## 何时使用此技能

当用户：
- 想要安装、创建、管理技能
- 想要查看技能总览或健康报告
- 想要推送技能到 GitHub
- 想要归档过期技能
- 想要检查技能更新

---

## 核心能力

| 能力域 | 说明 |
|--------|------|
| **输入管理** | 外部安装 (find-skills + skill-vetter) + 内部原创 (skill-creator) |
| **输出管理** | 推送 GitHub (skill-uploader) + 归档备份 |
| **层级管理** | 金字塔三层分类 (预定义 + 动态归类) |
| **生命周期** | 四阶段自动流转 (量化指标驱动) |
| **版本控制** | GitHub 远程同步 + 更新检测 |

## 主菜单交互

当用户调用此技能时，展示以下菜单：

```
┌─────────────────────────────────────────────────────────────┐
│ 你好！我是你的技能管家。今天想做什么？                        │
│                                                             │
│ [1] 📥 安装新技能    [2] ✨ 创建新技能                       │
│ [3] 📊 查看技能总览  [4] 🔄 检查更新                         │
│ [5] 📤 推送技能      [6] 🗑️ 归档技能                        │
│ [7] 📈 技能健康报告  [8] ⚙️ 设置                            │
└─────────────────────────────────────────────────────────────┘
```

根据用户选择，执行对应操作：

### [1] 📥 安装新技能
1. 询问用户想要什么功能的技能
2. 调用 `find-skills` 搜索相关技能
3. 展示搜索结果，让用户选择
4. 调用 `skill-vetter` 进行安全审查
5. 展示审查结果，确认是否安装
6. 执行安装命令：`npx skills add <package> -g -y`
7. 运行 `python scripts/scan_skills.py` 更新技能列表
8. 询问用户确认层级分类

### [2] ✨ 创建新技能
1. 调用 `skill-creator` 引导用户创建技能
2. 创建完成后运行 `python scripts/scan_skills.py` 更新列表
3. 自动分类层级（可让用户确认）

### [3] 📊 查看技能总览
1. 运行 `python scripts/scan_skills.py` 确保数据最新
2. 运行 `python scripts/show_pyramid.py` 展示金字塔视图
3. 询问用户是否需要查看某个技能的详细信息

### [4] 🔄 检查更新
1. 扫描所有 `source_type=github` 的技能
2. 对每个技能检查远程是否有更新
3. 展示可更新的技能列表
4. 询问是否执行更新

### [5] 📤 推送技能
1. 列出 `source_type=local` 且 `lifecycle=成熟稳定期` 的技能
2. 让用户选择要推送的技能
3. 调用 `skill-uploader` 打包上传
4. 更新 `source_type` 为 `github`
5. 状态流转到 `上线迭代期`

### [6] 🗑️ 归档技能
1. 列出 `lifecycle=下线归档期` 的技能
2. 让用户确认要归档的技能
3. 执行归档操作（备份 + 删除）

### [7] 📈 技能健康报告
1. 运行 `python scripts/scan_skills.py`
2. 生成健康报告，包括：
   - 需要关注的技能（过期、有更新、萌芽期）
   - 状态良好的技能统计
   - 优化建议

### [8] ⚙️ 设置
1. 查看当前配置
2. 修改设置

## 金字塔层级分类

详见 `references/pyramid.md`

```
        ▲
       /│\        L3-高阶决策技能 (顶层)
      / │ \       规划、架构、治理、策略
     /  │  \
    /───┼───\     L2-领域核心技能 (中层)
   /    │    \    岗位独有的专业技能
  /     │     \
 /──────┼──────\  L1-公共技能 (底层)
/       │       \ 工具使用、流程执行、沟通协作
```

**分类规则**：
1. 先匹配 L3 高阶决策关键词
2. 再匹配 L2 领域核心（按子分类）
3. 默认归为 L1 公共技能

## 生命周期管理

详见 `references/lifecycle.md`

| 阶段 | 判断标准 | 图标 | 建议动作 |
|------|---------|------|---------|
| **需求萌芽期** | `invocation_count == 0` | 🟡 | 评估试用、完善文档 |
| **成熟稳定期** | `invocation_count >= 2` AND NOT pushed | 🟢 | 推送 GitHub、固化实践 |
| **上线迭代期** | `pushed == True` OR `invocation_count >= 5` | 🔵 | 检查更新、收集优化 |
| **下线归档期** | `last_invoked < 90天` OR `user_marked` | 🔴 | 归档、备份删除 |

**状态流转（单向线性）**：
```
需求萌芽期 → 成熟稳定期 → 上线迭代期 → 下线归档期
```
