# 生命周期管理规则

> 技能的四阶段生命周期定义

---

## 状态定义

| 阶段 | 判断标准 | 图标 | 建议动作 |
|------|---------|------|---------||
| **需求萌芽期** | `invocation_count == 0` OR `in_pending_list` | 🟡 | 评估试用、完善文档 |
| **成熟稳定期** | `invocation_count >= 2` AND NOT pushed | 🟢 | 推送 GitHub、固化实践 |
| **上线迭代期** | `pushed == True` OR `invocation_count >= 5` | 🔵 | 检查更新、收集优化 |
| **下线归档期** | `last_invoked < 90天` OR `user_marked` | 🔴 | 归档、备份删除 |

---

## 状态流转（单向线性）

```
┌─────────────┐     首次调用      ┌─────────────┐
│ 需求萌芽期   │ ───────────────► │ 成熟稳定期   │
│ (inv=0)     │                  │ (inv≥2)     │
│ 🟡          │                  │ 🟢          │
└─────────────┘                  └──────┬──────┘
                                        │
                               推送/调用≥5
                                        │
                                        ▼
                                 ┌─────────────┐
                                 │ 上线迭代期   │
                                 │ (pushed/inv≥5)
                                 │ 🔵          │
                                 └──────┬──────┘
                                        │
                               90天未调用/手动标记
                                        │
                                        ▼
                                 ┌─────────────┐
                                 │ 下线归档期   │
                                 │ (deprecated) │
                                 │ 🔴          │
                                 └─────────────┘
```

**重要**：这是正向流转的生命周期，**不会回流**。

## 触发条件表

| 当前状态 | 触发条件 | 目标状态 |
|---------|---------|---------||
| 需求萌芽期 | `invocation_count >= 1` | 成熟稳定期 |
| 成熟稳定期 | `pushed == True OR invocation_count >= 5` | 上线迭代期 |
| 上线迭代期 | `last_invoked < 90天前 OR user_marked` | 下线归档期 |

---

## 各阶段详细说明

### 🟡 需求萌芽期

**进入条件**：
- 新安装的技能（`invocation_count == 0`）
- 在待创建列表中的技能

**特征**：
- 尚未验证其实用性
- 可能只是尝试性安装

**建议动作**：
- 评估试用
- 完善文档
- 如果不需要，及时删除

### 🟢 成熟稳定期

**进入条件**：
- `invocation_count >= 2`（至少调用过2次）
- 尚未推送到 GitHub

**特征**：
- 已验证其实用性
- 使用频率适中
- 价值已确认

**建议动作**：
- 推送到 GitHub 固化
- 完善文档和测试
- 分享给他人

### 🔵 上线迭代期

**进入条件**：
- `pushed == True`（已推送到 GitHub）
- 或 `invocation_count >= 5`（调用超过5次）

**特征**：
- 已公开分享
- 或高频使用
- 进入稳定维护阶段

**建议动作**：
- 定期检查更新
- 收集优化建议
- 保持版本同步

### 🔴 下线归档期

**进入条件**：
- `last_invoked < 90天前`（超过90天未使用）
- 或 `user_marked == True`（用户手动标记）

**特征**：
- 长期未使用
- 可能已被替代
- 或用户明确不再需要

**建议动作**：
- 归档备份
- 删除释放空间
- 保留配置记录

## 生命周期检查脚本

运行 `python scripts/check_lifecycle.py` 可以：
1. 扫描所有技能
2. 检查是否需要状态变更
3. 输出变更建议列表
