## 输入管理

### find-skills（技能搜索）

**用途**：搜索外部技能

**调用场景**：
- 用户选择"安装新技能"
- 用户提供技能需求描述

**返回**：
- 匹配的技能列表
- 安装命令

### skill-vetter（安全审查）

**用途**：审查外部技能安全性

**调用场景**：
- 用户选择要安装的技能后
- 在执行安装之前

**返回**：
- 安全审查结果
- 风险提示
- 是否建议安装

### skill-creator（技能创建）

**用途**：引导创建新技能

**调用场景**：
- 用户选择"创建新技能"
- 用户需要定制化技能

**返回**：
- 创建的技能目录
- 技能模板文件

---

## 内部管理

### 分类脚本（classify_skill.py）

**用途**：智能分类技能层级

**逻辑**：
1. 匹配 L3 高阶决策关键词
2. 匹配 L2 领域核心子分类
3. 默认归为 L1 公共技能

### 生命周期脚本（check_lifecycle.py）

**用途**：检查和更新生命周期状态

**逻辑**：
- 需求萌芽期 → 成熟稳定期：首次调用
- 成熟稳定期 → 上线迭代期：推送/调用≥5次
- 上线迭代期 → 下线归档期：90天未调用/手动标记

---

## 输出管理

### skill-uploader（技能推送）

**用途**：打包上传技能到 GitHub

**调用场景**：
- 用户选择"推送技能"
- 技能进入"成熟稳定期"

**返回**：
- 推送结果
- GitHub URL

### archive_skill.py（归档脚本）

**用途**：备份和删除过期技能

**功能**：
- 备份技能到归档目录
- 从列表中移除
- 可选删除文件

---

## 数据流

```
安装流程:
用户需求 → find-skills 搜索 → skill-vetter 审查 → 安装命令执行
    → scan_skills.py 扫描 → classify_skill.py 分类 → 保存到 skill-usage.json

创建流程:
用户需求 → skill-creator 创建 → scan_skills.py 扫描 → 自动分类 → 保存

推送流程:
选择技能 → 检查状态 → skill-uploader 打包 → 推送 GitHub → 更新 source_type

归档流程:
选择技能 → archive_skill.py 备份 → 删除文件 → 从列表移除
```

---

## 最佳实践

1. **安装外部技能**：始终先审查再安装
2. **创建原创技能**：使用 skill-creator 标准化流程
3. **定期健康检查**：每周运行 check_lifecycle.py
4. **及时归档**：90天未用的技能及时清理
5. **版本同步**：定期运行 check_updates.py
